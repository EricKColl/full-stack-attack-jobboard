# Arquitectura web de frontend y usabilidad - Producto 1

## 1. Arquitectura web frontend (descripción simple)
En este producto se desarrolla una arquitectura frontend básica compuesta por:
- **HTML5** para la estructura de las páginas
- **CSS3 + Bootstrap** para estilos, maquetación responsive y componentes visuales
- **JavaScript** para la lógica de interacción con el usuario y manipulación del DOM

La aplicación se organiza en varias pantallas HTML independientes, cada una con su script JS específico.  
Los datos se centralizan en un único archivo `datos.js`, que exporta arrays y objetos simples para simular el funcionamiento de una aplicación real.

No existe backend ni base de datos en este producto, por lo que toda la información se gestiona en memoria.

## 2. Usabilidad (descripción simple)
Para mejorar la usabilidad de la aplicación se aplican los siguientes criterios:
- Navegación clara y consistente en todas las pantallas
- Formularios con etiquetas visibles y comprensibles
- Diferenciación visual entre ofertas y demandas
- Mensajes de feedback en acciones importantes (login, alta, borrado)
- Diseño responsive para mejorar la visualización
- Distribución visual simple y ordenada para facilitar el aprendizaje del usuario

## 3. Mejores prácticas UX/UI aplicadas
- Consistencia visual entre pantallas
- Jerarquía visual en títulos, botones y contenidos
- Uso de colores con significado funcional
- Minimización de pasos para realizar acciones principales
