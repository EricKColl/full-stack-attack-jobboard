# Checklist de mockup - Producto 1 (JobConnect)

## 1. Introducción
Este documento recoge una checklist de verificación del mockup del proyecto **JobConnect** (Producto 1), con el objetivo de comprobar que el diseño propuesto cumple criterios básicos de:

- estructura
- coherencia visual
- usabilidad
- cobertura de las interfaces requeridas

> Nota: El mockup visual (imagen o PDF) se adjunta aparte en la documentación/evidencias del proyecto.

---

## 2. Checklist general del mockup

## 2.1 Cobertura de interfaces requeridas
- [x] El mockup incluye la **pantalla Dashboard**
- [x] El mockup incluye la **pantalla Login**
- [x] El mockup incluye la **pantalla Gestión de ofertas y demandas**
- [x] El mockup incluye la **pantalla Gestión de usuarios**

---

## 2.2 Estructura y organización visual
- [x] La interfaz tiene una estructura clara por bloques/secciones
- [x] Se distingue visualmente la navegación principal
- [x] Los formularios están organizados de forma lógica
- [x] Las tablas/listados tienen columnas identificables
- [x] Existe una jerarquía visual básica (títulos, contenido, acciones)

---

## 2.3 Consistencia entre pantallas
- [x] Se mantiene una **navbar común** en todas las pantallas
- [x] Se usa una estructura visual similar entre páginas
- [x] Los botones principales mantienen estilo coherente
- [x] Los formularios comparten un diseño uniforme
- [x] La navegación permite cambiar de pantalla de forma intuitiva

---

## 2.4 Usabilidad (criterios básicos)
- [x] Los textos principales son comprensibles
- [x] Los campos del formulario tienen etiquetas claras
- [x] Las acciones principales son visibles (guardar, eliminar, iniciar sesión)
- [x] Se prevé feedback visual para errores y acciones correctas
- [x] El usuario puede entender rápidamente qué hace cada pantalla

---

## 2.5 Dashboard (mockup)
- [x] Se muestran tarjetas o bloques resumen
- [x] Se diferencia la visualización de ofertas y demandas
- [x] Se incluye una zona para publicaciones resumidas
- [x] Se contempla el número total de usuarios/ofertas/demandas

---

## 2.6 Pantalla Login (mockup)
- [x] Se incluye formulario con email y contraseña
- [x] Se contempla botón de acceso
- [x] Se reserva espacio para mensajes de validación o error

---

## 2.7 Gestión de ofertas y demandas (mockup)
- [x] Se incluye formulario de alta de publicación
- [x] El formulario contempla tipo (oferta/demanda)
- [x] Se incluye tabla/listado de publicaciones
- [x] Se contempla acción de eliminar
- [x] Se diferencia visualmente el tipo de publicación

---

## 2.8 Gestión de usuarios (mockup)
- [x] Se incluye formulario de alta de usuario
- [x] Se incluye tabla/listado de usuarios
- [x] Se contempla acción de eliminar usuario
- [x] Se muestra información clave (nombre, email, rol)

---

## 2.9 Buenas prácticas de separación (HTML/CSS/JS)
- [x] El diseño está pensado para separar estructura (HTML), estilo (CSS) y lógica (JS)
- [x] Los componentes visuales son compatibles con Bootstrap
